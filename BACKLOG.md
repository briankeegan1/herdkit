## Reliability / safety

- 🔜 **Layout reconciler — engine must observe the live pane layout before mutating it (eyes-on-layout)** *(user insight 2026-07-02)* — every pane-mutating path (`cmd_reload`, `coordinator.sh`, future `herd pane` subcommands) acts on beliefs — the `.herd-panes` registry (stale/poisonable, e.g. Path B overwrote it with stray-tab IDs) and one-shot geometric neighbor guesses — instead of observing reality; a herd reload built standalone tabs and a rerun split a duplicate backlog-view pane. The engine already has the eyes: `herdr pane list` + `herdr pane process-info` can classify every pane in a tab by the script it runs (`backlog-view.sh`, `agent-watch.sh`/`herd-watch.sh`, claude agent, bare shell). Build a shared layout-snapshot helper in `scripts/herd` (scan tab → roles → reconcile against desired layout: adopt panes already serving a role, create only what is missing, close/flag duplicates, rewrite the registry from the observed scan on every mutation) and make `cmd_reload` + `coordinator.sh` consume it; hermetic tests with faked pane/process-info JSON covering stale-registry, duplicate-viewer, and missing-pane cases. Complements the in-flight reload-anchor-no-agent fix (a scan-before-mutate addendum was sent to that builder) and the sandbox-consumer sim item, which verifies this visually. Sequence after `reload-anchor-no-agent` merges. **Geometry mechanics (learned 2026-07-02 live-layout fix):** (1) Desired-geometry spec: backlog viewer occupies the full-height left column; right column = coordinator pane over watch pane, split at ratio 0.72 (`coordinator.sh:84`). (2) `herdr pane move <pane> --tab <t> --split down --target-pane <p> --ratio N` re-parents a live pane without killing its process, but a same-tab move is a no-op (returns `changed:false reason:same_tab`) — the working recipe is bounce out via `--new-tab` then move back in with `--target-pane`; the temp tab auto-closes when emptied. (3) Width-sensitive viewers (`backlog-view.sh` re-renders only on mtime/banner change) need a repaint nudge after geometry changes — `touch`-ing `BACKLOG.md` suffices since the render loop reads `tput cols` per frame.
- 🔜 **Sandbox consumer — a dummy repo + herdr workspace for live workflow simulation, especially pane/tab behavior** — every painful bug on 2026-07-02 was integration-level and VISUAL (review pane blank from `claude -p` buffering; reload's watcher pane sitting at a bare prompt while the process ran detached; spawns landing in whichever workspace had focus; stranded review/resolve tabs; cp1252 silently truncating the control room) — the hermetic test suite structurally cannot see any of these because it stubs herdr and never renders a pane. Build a live simulation rig: (1) a dedicated sandbox repo hosted on GitHub (e.g. `herdkit-sandbox`: a tiny real app + seeded BACKLOG.md) with a `herd sim init`-style fixture generator that deterministically resets it (fresh branch, clean worktrees dir, seeded issues) so every run starts identical — reuse the conventions from the shipped cross-repo loop simulation (PR #24, scripts/herd/sim); (2) a dedicated herdr workspace ('sandbox') where scenarios spin up a REAL control room so pane/tab experiments never touch real projects; assert placement/layout/labels/teardown via `herdr tab list` / `pane list` / `agent list` JSON at each checkpoint; (3) scripted end-to-end scenarios: init → control-room launch → spawn builder → PR → review gate → merge → teardown, with a STUB builder agent (deterministic tiny change, no model call) so runs are fast/cheap, plus an opt-in real-model smoke mode; (4) VISUAL confirmation: capture pane contents (herdr capture API if available, else launch a terminal window and take OS screenshots, e.g. macOS `screencapture` of the herdr window) at checkpoints; keep golden screenshots and have a vision-capable `claude -p` judge assert observable truths ('3 panes present', 'watch console painting rows', 'review pane streaming text') rather than pixel-diffing; (5) fault-injection scenarios reproducing 2026-07-02's bug classes: kill the watcher mid-review (expect INFRA retry not cached BLOCK), never-committed backlog file, `PYTHONIOENCODING=ascii` to simulate cp1252, missing optional deps (glow), duplicate-watcher race, focused-workspace misdirection — assert LOUD failures, never silent truncation; - ✅ **AUDIT (research pass): end-to-end flow redundancy, speed-ups, and gate necessity/correctness** — **Deliverable:** a findings doc (`docs/flow-audit.md`) + a RANKED list of concrete follow-up items, not a single code change; this is the umbrella under which the six spun-out efficiency levers below sit. **Why (surfaced 2026-07-02 to 2026-07-03):** the sha-cache fix (a held PR re-ran the full healthcheck every ~90s tick and manufactured its own transient code-error→flaky on PR #65) is likely one symptom of a broader pattern — the whole pipeline probably repeats work and runs gates that are redundant, unnecessary for some diffs, or sequenced sub-optimally. **Scope — every stage end-to-end:** (1) backlog/scribe — per-item git pulls, drainer spawn/teardown churn; (2) lane spawn — worktree + SHARE_LINKS symlinks, multi-kilobyte argv prompts; (3) build — isolated builders re-deriving context from files; (4) WATCHER TICK LOOP (`agent-watch.sh`) — how many `gh`/`herdr`/`git` subprocesses it forks per tick (latency + rate-limit + cost), what it recomputes every tick vs could cache, and whether the tick cadence itself is right; (5) HEALTHCHECK gate — serialization (`HEALTH_CONCURRENCY=1`: always necessary, or can worktree-isolated suites parallelize?), retry-before-red, and the now-added sha-cache; (6) REVIEW gate — per-PR Opus cost, whether trivial/docs/test-only diffs need it at all (ties to the risk-tiered-review item), and whether review + healthcheck overlap; (7) auto-refix rounds + wake verification; (8) merge re-verification; (9) reap — worktree remove, tab close, backlog reap, orphan sweep. **Look for four things at each stage:** (a) REDUNDANT work (repeated subprocess calls, re-reads, re-runs of unchanged inputs); (b) SPEED-UPS (caching, parallelization, better sequencing, cheaper models/tools); (c) UNNECESSARY gates — anything duplicating GitHub's own branch-protection/required-checks or adding latency without safety value; (d) gate ISSUES / correctness — races, false-reds, the manufactured-transient class, gates that can cache a wrong verdict. **Method:** token/cost instrumentation is ✅ shipped (commit e2a0ae5 — `herd cost`; per-builder/review accounting); add latency instrumentation as the audit proceeds, then map the flow, then rank findings; each significant finding becomes its own item; validate any flow change on the sandbox-consumer sim rig before shipping. **Composes with:** the six efficiency levers below (its children), the herdkit-vs-harness EPIC in `## Someday / Deferred` (shares the instrumentation precursor), and the sandbox-consumer rig (its safe test bed). **DELIVERABLE SHIPPED:** `docs/flow-audit.md` landed as PR #77 (commit 14f7e42);
