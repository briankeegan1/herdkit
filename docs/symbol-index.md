> **Honest scope — a heuristic, not a parser.** This is a bash-native token scan, so:
> • It cannot tell which same-named definition a call binds to — a name defined in several
>   files (e.g. `_backend_update_state`, one per backend) lists every def site, and its

>   built by dynamic dispatch (`"$cmd" ...`), and tokens inside heredoc bodies, quoted
>   strings, or trailing `# comments`, may be missed or over-counted.
> • Same-file calls are omitted by design — this indexes CROSS-file reach only.
