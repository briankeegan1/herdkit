> Read-only analysis (HERD-43). Audited at engine commit `cc59352`, starting from
> [`docs/codemap.md`](codemap.md) (module roles, source edges, config-key wiring) rather than a
> blind tree walk. Conclusions drawn on `claude-fable-5`. Three lenses: **efficiency**,

## Lens 1 — Efficiency

### Baseline and measurement caveats

- `herd cost` (price table `scripts/herd/cost.sh:87-98`; cache multipliers `:82-84`): opus-4-8
  $5/$25 per Mtok, sonnet-4-6 $3/$15, haiku-4-5 $1/$5. **Opus→sonnet is a ~40% rate cut, not the
  5× intuition** — both tiers moved; every routing estimate below uses 0.6×.
- Component split: builder $5.22/PR, review $1.01/PR (measured).
- The review figure **undercounts structurally** (`cost.sh:24-26`): only reviews on the sha that
  actually merged are captured. Reviews on superseded shas (every refix round) and on PRs that
  BLOCKed and never merged are invisible. The true review lever is larger than 16% of spend.
- Builder cost is **cache-read dominated**: PR #150 spent $13.05 on its builder, of which 16.0 M
  cache-read tokens ≈ $8.00 — i.e. the cost of a PR scales with (turns × context size), not with
  code produced. Session length is the deepest builder lever.

### E1. Tiered review is built, tested, sha-cached — and switched off

- **Finding:** `REVIEW_ESCALATE_GLOB=""` in the dogfood config means the classifier never runs:
  the tiering block is skipped entirely and every PR gets the full `$MODEL_REVIEW` (Opus) review.
- **Evidence:** gate short-circuit `scripts/herd/agent-watch.sh:573`; classifier
  `_classify_review_tier` `:423-439` (docs/tests-only → SKIP `:430`, glob match → STRONG `:432`,
  `REVIEW_ESCALATE_MAXFILES` breach → STRONG `:434-436`, else CHEAP `:438`; empty diff fails safe
  to STRONG `:426-427`); sha-keyed memoization `:445-452`; SKIP recorded as
  `source=skipped-low-risk` PASS before consuming a reviewer slot `:575-584`; CHEAP model pick
  `:585`; evidence-triggered re-escalation to Opus after failed refix rounds `:593-595`; model
  handoff to the reviewer via `HERD_REVIEW_MODEL` `:517-521` → `scripts/herd/herd-review.sh:110`.
- **Grounding:** classifying the last 45 merged PRs against an engine-critical glob routes
  **13/45 (29%) CHEAP + 1/45 SKIP** — the cheap cohort is viewers, backends, fleet helpers, sim,
  and preflight work (PRs #110 #113 #127 #129 #133 #134 #138 #140 #143 #148 #149 #153), which is
  exactly the intended low-risk surface.
- **Unlock:** set, in `.herd/config`:
  `REVIEW_ESCALATE_GLOB='^bin/herd|^scripts/herd/(agent-watch|driver|healthcheck|herd-config|herd-review|herd-quick|herd-feature|journal|spawn)|^\.herd/'`
- **Estimated impact:** measured arithmetic: 29% × $1.01 × 0.4 (CHEAP) + 2% × $1.01 (SKIP)
  ≈ **$0.14/PR**; true saving likely **$0.15–0.30/PR** because uncounted superseded-sha reviews
  re-run per refix round and would tier identically. Risk low: misclassification fails toward
  STRONG, and a cheap PASS that survives two failed refix rounds is re-reviewed on Opus anyway.
- **Suggested backlog item:** `config: set REVIEW_ESCALATE_GLOB — engine-critical glob to activate tiered review`

### E2. `MODEL_QUICK=claude-opus-4-8` sends every builder through the top tier

- **Finding:** herdkit's dogfood config routes the quick lane — which on this repo *is* the engine
  lane — to Opus, so 100% of builder spend (84% of total) is at the top rate. The deterministic
  step-up machinery that would make a cheaper default safe already exists and is already
  configured.
- **Evidence:** `.herd/config` sets `MODEL_QUICK="claude-opus-4-8"` (with the comment "quick IS
  the engine lane"); model pick `scripts/herd/herd-quick.sh:59`; `MODEL_ESCALATE_GLOB` step-up to
  `$MODEL_FEATURE` `:60-69` (already set to `bin/herd|scripts/herd/agent-watch|herd-review|cmd_reload`);
  spawn `:176`. Same pattern in `herd-feature.sh:52-61`.
- **Unlock:** `MODEL_QUICK="claude-sonnet-4-6"` + widen `MODEL_ESCALATE_GLOB` to the same
  engine-critical list as E1 (add `driver|healthcheck|herd-config|journal|spawn|herd-quick|herd-feature`).
  Task text touching critical surfaces then deterministically builds on Opus; viewer/backend/
  fleet/docs tasks build on sonnet.
- **Estimated impact:** builder $5.22/PR × 0.4 saving on routed PRs: at the 29% low-risk share
  ≈ **$0.61/PR**; if half of tasks route ≈ **$1.05/PR**. Risk medium: the config comment records
  a real sonnet misbuild on this surface, and a misbuild's refix rounds (new review + health suite
  per push, E5) can eat the margin — which is why the glob, not `TOKEN_MODE`, is the right lever.
- **Suggested backlog item:** `config: MODEL_QUICK → sonnet with widened MODEL_ESCALATE_GLOB (deterministic Opus step-up)`

### E3. `TOKEN_MODE=eco` — assessed, not recommended for the engine repo

- **Finding:** eco flips *defaults only* (explicit `MODEL_*` keys always win): coordinator/feature
  → sonnet, quick/scribe/research/resolver → haiku, review → sonnet. On herdkit, where every
  explicit `MODEL_*` key is set, flipping to eco alone would change **nothing**; making it bite
  requires deleting the explicit keys, which removes the deterministic escalate guards' floor.
- **Evidence:** eco block `scripts/herd/herd-config.sh:230-238` (review flip `:236`); precedence
  documented in `templates/capabilities.tsv` row `TOKEN_MODE`.
- **Unlock / recommendation:** keep eco as the **consuming-project** starter posture (it already
  is — `templates/config.example:78-83` ships eco-leaning defaults); on the engine repo, get the
  same dollars via E1+E2's targeted globs. Gross ceiling if applied anyway: builder to sonnet
  (−$2.09/PR) + review to sonnet (−$0.40/PR) ≈ −$2.50/PR, at the cost of un-guarded tiers on the
  repo whose bugs cost the most to ship.
- **Estimated impact:** $0 as-configured (explicit keys mask it); documented so nobody flips it
  expecting savings. Risk of the "delete the keys" variant: high.
- **Suggested backlog item:** `docs: TOKEN_MODE=eco is masked by explicit MODEL_* keys — document the precedence trap`

### E4. Coordinator per-session context tax — post-#157 state and the next cut

- **Finding:** PR #157 cut the rendered skill from ~74.8 KB (~18.7 K tokens) to ~38.7 KB
  (~9.7 K tokens) by compacting the inline capabilities catalog (~50 KB → ~14 KB). What remains
  per session: the ~14 KB inline index, an every-invocation live-state read set (`herd backlog`,
  `git worktree list`, list-agents, `herd link list`, `cat .herd/deps`), and a mandated
  memory read+write cycle. Notably, the skill **never references `docs/codemap.md`** — the
  freshly-shipped map (HERD-36) built precisely to replace exploratory re-reads.
- **Evidence:** template `templates/coordinator.md.tmpl` (352 lines / 21,925 bytes; on-invocation
  reads `:15-34`; capabilities injection `:313`; every-session memory section `:327-341`);
  render-time index compaction in `bin/herd` `render_skill` (PR #157, commit `99c9f37`).
- **Unlock:** (a) demote the inline index to a two-line pointer at
  `docs/capabilities-overview.md` + `templates/capabilities.tsv` (the detail pointer already
  exists post-#157 — the index duplicates it); (b) add `docs/codemap.md` to the skill as the
  on-demand repo map so coordinator research reads one 8 KB file instead of exploring.
- **Estimated impact:** ~3.5 K tokens/session off the skill load. Direct $ is small (coordinator
  spend is not even in the $6.23/PR figure — see E6), but every token of standing tax is re-read
  each turn at cache-read rates for the whole session; the operative unlock is **context
  headroom** — later compaction, longer useful sessions.
- **Suggested backlog item:** `coordinator: point the skill at docs/codemap.md and demote the inline capabilities index to a pointer`

### E5. Refix-loop economics: each bounce is a full review + health suite

- **Finding:** with `REVIEW_AUTOFIX=true` (dogfood), a BLOCK verdict re-wakes the builder in its
  existing session (cheap — extra turns, no new session), but **every re-pushed commit triggers a
  fresh full review session and a fresh health suite** (new sha → re-dispatch), up to
  `REFIX_MAX_ROUNDS=3`, with the final round's review forced to Opus. Combined with cache-read
  dominance (a long session's every turn re-reads the whole context at 0.10× input), a stubborn
  PR's cost grows superlinearly — the $11–14 tail PRs (#123 #142 #150) are the distribution's
  heavy tail.
- **Evidence:** bounce handler `scripts/herd/agent-watch.sh:1109-1189` (re-task prompt to the live
  pane `:1157-1173`); round cap `:1132`; escalation arming `:1143` consumed at `:593-595`;
  cache-read arithmetic from `herd cost --pr 150` (16.0 M cache-read tokens ≈ $8.00 of $13.05).
- **Unlock:** attribute cost per refix round in the journal (the data exists in transcripts;
  `cost.sh` already prices sessions) so the coordinator can see "this PR has burned 2 rounds ×
  $1.50" and intervene — hand-fix, re-spec, or kill — instead of paying round 3 blind.
- **Estimated impact:** visibility first; the actionable saving is the E1 tiering applied to
  refix re-reviews (uncounted today) plus earlier human intervention on tail PRs. Even one
  avoided round-3 cycle on a tail PR is ≈ $1–2.
- **Suggested backlog item:** `cost: journal per-refix-round cost attribution so tail PRs surface before round 3`

### E6. Cost measurement blind spots understate spend and skew the levers

- **Finding:** three cohorts are invisible to `herd cost`: reviews on superseded shas, reviews on
  never-merged PRs, and the coordinator/scribe/researcher sessions (a summer exists —
  `_COST_FULL_PY` — but their spend is not folded into the per-PR baseline). Decisions made on
  the $6.23 figure therefore see 84/16 builder/review when the real review share is higher.
- **Evidence:** documented gap `scripts/herd/cost.sh:24-26`; reviewer fingerprint classification
  `:229-231` (merged-worktree transcripts only); component summers `:224-262`.
- **Unlock:** scan review transcripts for *all* shas of a merged PR and for closed-unmerged PRs;
  journal a `cost` event per review dispatch rather than only at `do_merge`.
- **Estimated impact:** $0 saved directly; makes E1/E2's real return measurable instead of
  estimated. Cheap insurance against optimizing the wrong lever.
- **Suggested backlog item:** `cost: count superseded-sha and blocked-PR review sessions in herd cost`

### E7. Review/health concurrency — a latency lever, not a cost lever (leave as-is)

- **Finding:** `REVIEW_CONCURRENCY=2` caps parallel reviewer sessions (queue, don't skip) and
  `HEALTH_CONCURRENCY=1` serializes health suites. Neither changes per-PR token cost — only
  wall-clock. The health mutex is correctness-driven: all worktrees share one git object store,
  and overlapping suites race on `.git` locks and paint false reds. The sha-keyed health cache
  already prevents re-runs on unchanged shas.
- **Evidence:** review gate `scripts/herd/agent-watch.sh:599`; health slot `:2191-2194`; queued
  verdict semantics `:2250`; sha-cache `:2201-2207`; 4 s tick `:3099`.
- **Unlock:** none needed — documented here so the "raise concurrency to save money" intuition
  dies. Raising `REVIEW_CONCURRENCY` raises burn *rate* and merge throughput, not efficiency.
- **Estimated impact:** $0; prevents a plausible-looking regression.
- **Suggested backlog item:** none (non-action).

### E8. Researcher round-trips — verified efficient (non-finding)

- **Finding:** the suspected "redundant researcher round-trips" do not exist. One persistent
  drainer session claims queued questions in-session (`research-step.sh next`), batches through
  them without restarting Claude per question, self-terminates on empty queue, and runs on
  sonnet. Per-question cost = drainer turns + its Explore subagents. The only standing overhead
  is the per-spawn `RULES` preamble in the builder lanes (~700 chars re-paid when spawns are
  >5 min apart, duplicated verbatim in `herd-quick.sh:154-157` and `herd-feature.sh`) — ≈ $0.001/PR,
  a maintenance smell rather than a cost item. (HERD-40 has since factored its new grounding
  preamble into a shared `herd_context_provision_preamble` helper — the right pattern; the rest of
  the rules text remains two verbatim copies.)
- **Evidence:** singleton check `scripts/herd/research.sh:66-77`; batching loop prompt `:80-101`;
  queue claim `scripts/herd/research-step.sh:28-52`; self-termination `:65-69`; model
  `research.sh:26` → `herd-config.sh:244`.
- **Suggested backlog item:** none (verified non-finding; record so it is not re-investigated).

## Lens 2 — Customizability

Sweep of `bin/herd` + `scripts/herd/*.sh` + `scripts/herd/backends/*` + templates against the
config surface (`templates/capabilities.tsv` kind=config rows, defaults in
`scripts/herd/herd-config.sh`). Ordered fork-blocking → tuning → per-machine → architectural.
Verified clean (checked, not findings): ports (all via `APP_PREVIEW_*` keys), temp paths (all
`${TMPDIR:-/tmp}`), Linear team id (resolved via `LINEAR_TEAM_ID`, `HERD` appears only in
comments), repo slug (`HERD_REPO` key; zero hardcoded `briankeegan1` in engine source).

### C1. Branch prefix `feat/` is hardcoded in the sole branch creator

- **Evidence:** `scripts/herd/new-feature.sh:16` (`BRANCH="feat/$NAME"`), used by both lanes
  (`herd-quick.sh:75`, `herd-feature.sh:68`). No key exists.
- **Why it forces a fork:** repos with branch-name policies (`feature/*`, ticket-key prefixes,
  protected-prefix CI) cannot adopt without editing engine source.
- **Unlock / impact:** new key `BRANCH_PREFIX` (default `feat/`). Per-project. One-line change +
  tsv row; removes a hard adoption blocker for policy-bound repos.
- **Suggested backlog item:** `engine: BRANCH_PREFIX config key — new-feature.sh hardcodes feat/`

### C2. The review-skip classifier's low-risk pattern is baked into logic

- **Evidence:** `scripts/herd/agent-watch.sh:430` — docs/tests-only means "every path matches
  `(\.md$)|(^tests/)`". The STRONG surface (`REVIEW_ESCALATE_GLOB`), max-files, and cheap model
  are all keys; the *skip* pattern alone is hardcoded.
- **Why it forces a fork:** `test/`, `__tests__/`, `spec/`, `_test.go`-beside-source, or
  `.rst`/`.adoc` docs conventions make the skip never fire (wasted Opus reviews — the exact cost
  this tier exists to cut) or mis-fire on repos with real code under a matching path.
- **Unlock / impact:** new key `REVIEW_LOWRISK_GLOB` (default = current pattern, behavior-neutral).
  Per-project. Prerequisite for shipping E1's tiering as a credible cross-project default.
- **Suggested backlog item:** `engine: REVIEW_LOWRISK_GLOB key for the review-skip classifier`

### C3. `REVIEW_MODEL_ESCALATED` is an orphan — referenced, defined nowhere

- **Evidence:** `scripts/herd/agent-watch.sh:595` (`${REVIEW_MODEL_ESCALATED:-claude-opus-4-8}`,
  also console string `:2993`). Absent from `herd-config.sh` and `capabilities.tsv`; every
  sibling tier (`MODEL_REVIEW`, `REVIEW_MODEL_CHEAP`) is a documented key.
- **Unlock / impact:** define + document it (default `claude-opus-4-8`). Per-project. Also fixes
  a hardcoded model name outside the config layer — the only one found in engine logic.
- **Suggested backlog item:** `engine: define + document REVIEW_MODEL_ESCALATED (orphan env var, hardcoded fallback)`

### C4. Four working env knobs are undocumented — configurable in code, invisible in the manifest

- **Evidence:** `REVIEW_EVIDENCE_ESCALATE_ROUNDS` (default 2) `agent-watch.sh:1047`;
  `DEAD_GRACE_MIN` (default 120 s) `agent-watch.sh:1916-1918`; `HERD_REVIEW_AGENT_TIMEOUT`
  (default 1800 s) + `HERD_REVIEW_AGENT_POLL` (default 5 s) `herd-review.sh:448-449`. None appear
  in `capabilities.tsv`, unlike siblings `DEP_POLL_MIN/MAX`, `DEP_STALE_TTL`.
- **Why it matters:** slow-CI projects need a longer dead-grace and review timeout; today they'd
  have to read the source to discover the knob exists. `herd config` can't set or validate them.
- **Unlock / impact:** document all four as keys. Per-project tuning; low effort.
- **Suggested backlog item:** `docs: materialize DEAD_GRACE_MIN, REVIEW_EVIDENCE_ESCALATE_ROUNDS, HERD_REVIEW_AGENT_TIMEOUT/POLL as config keys`

### C5. Watcher tick and orphan-sweep cadence are pure literals — no override at all

- **Evidence:** `scripts/herd/agent-watch.sh:3099` (`sleep 4`) and `:2730`
  (`_ORPHAN_SWEEP_INTERVAL=15`, ≈60 s). No `${VAR:-…}` indirection.
- **Why it matters:** the tick drives per-cycle `gh pr list`/`gh pr diff` traffic; a busy shared
  repo hitting GitHub rate limits, or CI wanting a slower cadence, has no lever.
- **Unlock / impact:** new key `WATCHER_TICK_SECS` (default 4). Per-project.
- **Suggested backlog item:** `engine: WATCHER_TICK_SECS config key (watcher loop is a sleep-4 literal)`

### C6. Per-machine values for the planned `.herd/config.local` split

- **Evidence:** native notifier `scripts/herd/driver.sh:67-70` (`osascript` → `notify-send`
  fallback, kill-switch `HERD_HEADLESS_NATIVE_NOTIFY` `:66`; Windows gets neither — log-only);
  fleet registry `scripts/herd/fleet.sh:25` (`${HERD_FLEET_FILE:-$HOME/.herd/fleet}`); brew/
  Caskroom + quarantine diagnostics `herd-preflight.sh:318,412,435` (guarded, mac-only).
- **Recommended split:** **per-project** (`.herd/config`, committed): C1–C5 keys, globs, models,
  ports, team ids. **Per-machine** (`.herd/config.local`, gitignored): `NOTIFY_CMD` (new key
  generalizing the notifier), `HERD_FLEET_FILE`/`HERD_FLEET_ROOM_DIR`, `HERD_DRIVER` (herdr
  presence is a box property), limit-hook opt-outs (`HERD_LIMIT_HOOK`).
- **Suggested backlog item:** `engine: .herd/config.local overlay — NOTIFY_CMD, fleet paths, HERD_DRIVER are per-machine`

### C7. The deepest coupling is the git host, not the multiplexer

- **Evidence:** `gh pr create` embedded in builder rule text (`herd-quick.sh:103,110`,
  `herd-feature.sh:98,105,117,139`, pointer prompt `herd-config.sh:757`, respawn prompt
  `agent-watch.sh:2096`) — partially abstracted via `PR_CREATE_CMD` — while the watcher's entire
  merge/review machinery calls `gh` directly throughout.
- **Assessment:** a GitLab/Bitbucket shop forks today. This is architectural (a `GIT_HOST`
  abstraction on the scale of the driver work), not a config key; flagged for the roadmap, not
  for this cycle.
- **Suggested backlog item:** `roadmap: git-host abstraction — the watcher is hard-bound to gh`
